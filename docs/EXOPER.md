# The Dashboard Philosophy

The dashboard should prioritize:

## 1. Clarity

Users should instantly see:

* market structure,
* bias,
* active opportunities,
* invalidation points,
* risk exposure,
* and system reasoning.

---

## 2. Transparency

Every trade should explain:

* WHY it exists,
* WHY it was rejected,
* WHY confidence is high or low,
* WHAT rules validated it,
* WHAT rules failed.

This builds trust.

## 3. Structure Enforcement

The UI itself should discourage:

* overtrading,
* impulsive behavior,
* emotional execution.

Example:

* clear rejection reasons,
* visible confluence scoring,
* cooldowns,
* session restrictions,
* risk warnings.
