# infrastructure/cloudflare

Provisions the Cloudflare zone settings, DNS records, and AWS origin
firewall rules that gate the eTradie platform behind Cloudflare.

## What it owns

* Cloudflare zone TLS posture (min TLS 1.2, TLS 1.3 on, always-https,
  strict SSL).
* Authenticated Origin Pulls (AOP) at the zone level.
* DNS records (`api.etradie.com`, `*.etradie.com`, etc.) that resolve
  to the EKS NLB hostname output by `infrastructure/cluster/`.
* AWS Security Group ingress rules pinning TCP/443 to Cloudflare's
  published IPv4/IPv6 ranges.

## What it does NOT own

* The AOP CA bytes - the operator writes those into the Vault path
  bootstrapped by `infrastructure/cluster/`. The chart's
  ExternalSecret then synthesises the in-cluster Secret.
* The NLB itself (provisioned by the AWS Load Balancer Controller in
  response to the edge-ingress Service annotation).
* Any Kubernetes manifest.

## Apply

Apply AFTER `infrastructure/cluster/` and AFTER ArgoCD has reconciled
the edge-ingress Service so the NLB exists and a stable hostname is
available for CNAME targets.

```bash
cd infrastructure/cloudflare
terraform init
terraform apply \
  -var environment=production \
  -var zone_id=<...> \
  -var origin_security_group_id=<sg-...> \
  -var-file=cloudflare-ranges.tfvars   # generated from deployments/cloudflare/ip-ranges/
```

The `cloudflare-ranges.tfvars` file is generated by the weekly CI job
`deployments/cloudflare/scripts/refresh-cloudflare-ips.sh` to keep
this module's input in sync with the helm chart's ConfigMap.
