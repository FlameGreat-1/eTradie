# Local dev TLS for the docker-compose `edge` profile

This directory holds a self-signed `localhost` certificate +
private key that the docker-compose `edge-ingress` service mounts
at `/etc/edge-ingress/certs/`. The files are **gitignored**; you
must generate them on first use.

Generate per-workstation:

```bash
mkdir -p deployments/edge-ingress/docker/certs
cd deployments/edge-ingress/docker/certs
openssl req -x509 -nodes -newkey rsa:2048 \
  -keyout localhost.key \
  -out    localhost.crt \
  -days 365 \
  -subj "/CN=localhost/O=etradie-local/C=US" \
  -addext "subjectAltName=DNS:localhost,IP:127.0.0.1"
chmod 600 localhost.key
```

Validate:

```bash
openssl x509 -in localhost.crt -noout -subject -dates -ext subjectAltName
```

The `make dev-certs` target only generates the dev Cloudflare AOP
CA + client cert (under `deployments/cloudflare/origin-pull/`); the
`localhost` TLS pair is intentionally NOT regenerated by the same
target so a developer cannot accidentally rotate their local
certs and leak the previous pair into git history.

## History purge

A previous revision of this repository accidentally committed a
plaintext `localhost.key`. The repository default branch has been
updated to remove the file, and the leaked key MUST be considered
compromised (`localhost` scope only; cannot affect any production
endpoint). To purge the leak from history:

```bash
git filter-repo --invert-paths \
  --path deployments/edge-ingress/docker/certs/localhost.key \
  --path deployments/edge-ingress/docker/certs/localhost.crt
git push --force-with-lease origin main
```

This is operator action; not automatable through chart / CI tooling.
